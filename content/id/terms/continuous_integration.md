## Definition

Integrasi Berkelanjutan (CI) adalah praktik DevOps kritis yang mengotomatisasi penggabungan perubahan kode dari banyak kontributor ke dalam satu proyek perangkat lunak. Dengan menjalankan pembangunan dan pengujian otomatis, CI membantu mendeteksi kesalahan lebih awal dan memastikan integritas kode.

### Summary

Praktik pengembangan perangkat lunak di mana pengembang secara rutin menggabungkan perubahan kode ke dalam repositori terpusat, yang memicu pembangunan dan pengujian otomatis.

## Key Concepts

- Pengujian Otomatis
- Kontrol Versi
- Otomatisasi Pembangunan
- Siklus Umpan Balik

## Use Cases

- Memvalidasi perubahan kode model ML
- Menjamin reproduktibilitas alur kerja data
- Mengintegrasikan cabang fitur baru

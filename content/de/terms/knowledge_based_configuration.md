## Definition

Dieser Ansatz verwendet Techniken der Constraint Satisfaction (Erfüllung von Nebenbedingungen) innerhalb einer Wissensbasis, um sicherzustellen, dass montierte Produkte alle technischen und kundenbezogenen Anforderungen erfüllen. Er verhindert ungültige Kombinationen durch Encodierung von Regeln.

### Summary

Ein automatisierter Prozess, der domänenspezifische Wissensbasen nutzt, um aus Benutzerbeschränkungen gültige Produktkonfigurationen zu generieren.

## Key Concepts

- Constraint Satisfaction (Nebenbedingungserfüllung)
- Produktmodellierung
- Regelbasierte Systeme
- Automated Reasoning (Automatisches Schlussfolgern)

## Use Cases

- Zusammenbau von individueller Computer-Hardware
- Auswahl von Softwarelizenzen und Funktionen
- Einrichtung von Industriemaschinen

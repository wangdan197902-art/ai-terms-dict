## Definition

La Integración Continua (CI) es una práctica crítica de DevOps que automatiza la integración de cambios de código de múltiples colaboradores en un único proyecto de software. Al ejecutar compilaciones y pruebas automatizadas

### Summary

Una práctica de desarrollo de software en la que los desarrolladores fusionan frecuentemente los cambios de código en un repositorio central, lo que desencadena compilaciones y pruebas automatizadas.

## Key Concepts

- Pruebas Automatizadas
- Control de Versiones
- Automatización de Compilaciones
- Bucle de Retroalimentación

## Use Cases

- Validar cambios en el código de modelos de ML
- Garantizar la reproducibilidad de las tuberías de datos
- Integrar nuevas ramas de características

## Definition

L'Integrazione Continua (CI) è una pratica fondamentale del DevOps che automatizza l'integrazione delle modifiche al codice provenienti da più contributori in un singolo progetto software. Eseguendo build e test automatizzati

### Summary

Una pratica di sviluppo software in cui gli sviluppatori uniscono frequentemente le modifiche al codice in un repository centrale, attivando build e test automatizzati.

## Key Concepts

- Test Automatizzati
- Controllo di Versione
- Automazione della Build
- Ciclo di Feedback

## Use Cases

- Convalida delle modifiche al codice dei modelli ML
- Garantire la riproducibilità delle pipeline dei dati
- Integrazione di nuovi branch di funzionalità

## Definition

REST API ช่วยให้เกิดการสื่อสารระหว่างไคลเอนต์และเซิร์ฟเวอร์โดยใช้การทำงานแบบไร้สถานะ (stateless) ผ่านโปรโตคอล HTTP เช่น GET, POST, PUT และ DELETE โดยจัดโครงสร้างทรัพยากรเป็น URI และใช้รูปแบบข้อมูลมาตรฐาน

### Summary

REST API (Representational State Transfer API) คือรูปแบบสถาปัตยกรรมสำหรับการออกแบบแอปพลิเคชันบนเครือข่ายที่พึ่งพาวิธีการมาตรฐานของ HTTP

## Key Concepts

- ความเป็นไร้สถานะ (Statelessness)
- วิธีการ HTTP (HTTP Methods)
- URI ที่อ้างอิงตามทรัพยากร (Resource-Based URIs)
- การแปลงสภาพเป็น JSON (JSON Serialization)

## Use Cases

- การสื่อสารระหว่างไมโครเซอร์วิส (Microservices communication)
- การเชื่อมต่อแบ็กเอนด์สำหรับแอปพลิเคชันมือถือ (Mobile app backend integration)
- การเข้าถึงข้อมูลจากบุคคลที่สาม (Third-party data access)

## Definition

โทเค็นแมกซิ่งเกี่ยวข้องกับการออกแบบอินพุตอย่างพิถีพิถันเพื่อใช้ประโยชน์จากหน้าต่างบริบท (context window) ของโมเดลให้เต็มศักยภาพ หรือเพื่อเพิ่มความหนาแน่นของความหมายของโทเค็นเพื่อให้ได้ประสิทธิภาพการทำงานที่ดีขึ้น ผู้ปฏิบัติอาจเติมข้อความส่วนเกิน (padding) หรือปรับโครงสร้างประโยคเพื่อให้โมเดลประมวลผลข้อมูลได้อย่างมีประสิทธิภาพสูงสุดภายใต้ข้อจำกัดของทรัพยากร

### Summary

กลยุทธ์การปรับแต่งในวิศวกรรมคำสั่ง (Prompt Engineering) ที่มีเป้าหมายเพื่อเพิ่มประสิทธิภาพสูงสุดหรือคุณภาพผลลัพธ์จากการใช้จำนวนโทเค็นเฉพาะเจาะจงในแบบจำลองภาษาขนาดใหญ่

## Key Concepts

- วิศวกรรมคำสั่ง (Prompt Engineering)
- หน้าต่างบริบท (Context Window)
- ความหนาแน่นเชิงความหมาย (Semantic Density)
- การปรับให้เหมาะสม (Optimization)

## Use Cases

- เพิ่มขีดความสามารถในการให้เหตุผลของโมเดลภาษาขนาดใหญ่ (LLM)
- ลดต้นทุนการใช้งาน API
- ทดสอบขอบเขตความสามารถของโมเดล

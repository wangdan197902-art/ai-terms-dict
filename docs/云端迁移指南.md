# 云端迁移指南

> 本文档说明如何将 AI Terms Dictionary 从本地环境迁移到云端部署。

## 1. 迁移前提条件

### 1.1 当前本地架构
- **站点生成**：Hugo Extended v0.164.0
- **搜索索引**：Pagefind v1.5.2
- **AI内容生成**：AGNES_LLM (agnes-2.0-flash)
- **数据源**：本地种子库 (200术语)
- **部署**：hugo server 本地预览

### 1.2 云端目标架构
- **站点托管**：Cloudflare Pages / GitHub Pages / Vercel
- **CI/CD**：GitHub Actions
- **数据源**：Wikipedia API / arXiv API（可选）
- **AI内容生成**：AGNES_LLM（保持不变）
- **搜索**：Pagefind（构建时索引，零运行时依赖）

## 2. 适配器架构

项目已预留适配器接口，迁移时只需实现云端适配器：

```
scripts/utils/
├── source_adapter.py      # 数据源适配器
│   ├── LocalSourceAdapter     # 本地种子库（已实现）
│   └── CloudSourceAdapter     # 云端API（预留接口）
├── deploy_adapter.py      # 部署适配器
│   ├── LocalDeployAdapter     # 本地预览（已实现）
│   ├── CloudflarePagesDeployAdapter  # Cloudflare Pages（预留）
│   └── GitHubPagesDeployAdapter      # GitHub Pages（预留）
└── agnes_client.py        # AI客户端（无需修改）
```

## 3. 迁移步骤

### Step 1: 实现云端数据源适配器（可选）

```python
        pass
```

### Step 2: 配置GitHub Actions
创建 `.github/workflows/build.yml`：
```yaml
name: Build and Deploy
on:
  push:
    branches: [main]
  schedule:
    - cron: '0 0 * * 0'  # 每周日自动更新

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Setup Hugo
        uses: peaceiris/actions-hugo@v3
        with:
          hugo-version: '0.164.0'
          extended: true
      - name: Setup Python
        uses: actions/setup-python@v5
        with:
          python-version: '3.11'
      - name: Install dependencies
        run: pip install -r requirements.txt
      - name: Generate AI content
        env:
          AGNES_LLM_API_KEY: ${{ secrets.AGNES_LLM_API_KEY }}
          AGNES_LLM_BASE_URL: ${{ secrets.AGNES_LLM_BASE_URL }}
          AGNES_LLM_MODEL: ${{ secrets.AGNES_LLM_MODEL }}
        run: python3 scripts/generate_definitions_batch.py
      - name: Translate content
        run: python3 scripts/translate_batch.py
      - name: Build content
        run: python3 scripts/build_content.py
      - name: Build site
        run: hugo --gc --minify
      - name: Build search index
        run: npx pagefind --site public
      - name: Deploy
        uses: nwtgck/actions-netlify@v3.0
        with:
          publish-dir: './public'
          production-branch: main
          github-token: ${{ secrets.GITHUB_TOKEN }}
          deploy-message: "Deploy from GitHub Actions"
        env:
          NETLIFY_AUTH_TOKEN: ${{ secrets.NETLIFY_AUTH_TOKEN }}
          NETLIFY_SITE_ID: ${{ secrets.NETLIFY_SITE_ID }}
```

### Step 3: 配置生产环境变量
更新 `config/production/hugo.toml`：
```toml
baseURL = "https://ai-terms-dict.netlify.app/"
enableGitInfo = true
```

### Step 4: 配置Secrets
在GitHub仓库 Settings → Secrets 中添加：
- `AGNES_LLM_API_KEY`
- `AGNES_LLM_BASE_URL`
- `AGNES_LLM_MODEL`
- `NETLIFY_AUTH_TOKEN`（Netlify部署用）
- `NETLIFY_SITE_ID`（Netlify部署用）

## 5. 零迁移成本项

以下组件无需修改即可在云端运行：
- Hugo模板（`layouts/`）
- CSS/JS静态资源（`static/`）
- Python脚本（`scripts/`）
- AGNES_LLM客户端（`scripts/utils/agnes_client.py`）
- 内容构建脚本（`scripts/build_content.py`）
- Pagefind搜索索引（构建时生成）
- i18n字符串（`i18n/`）

## 6. 注意事项

1. **API密钥安全**：永远不要将API密钥提交到git，使用环境变量或Secrets
2. **构建时长**：GitHub Actions免费tier有6小时限制，200术语×6语种构建约5分钟
3. **Pagefind版本**：确保云端使用相同版本（v1.5.2）
4. **Hugo版本**：确保云端使用Hugo Extended（需要SCSS支持）
5. **缓存策略**：AGNES_LLM的本地缓存 `.cache/agnes/` 不应提交到git
